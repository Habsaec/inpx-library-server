# Эксплуатация inpx-library-server

## Windows (краткий чеклист)

1. Установите [Node.js](https://nodejs.org/) LTS (≥ 18).
2. Клонируйте или распакуйте проект, в корне: `npm install`.
3. Задайте переменные окружения или скопируйте `.env.example` в `.env` и укажите `LIBRARY_ROOT` и при необходимости `INPX_FILE`.
4. Запуск: `npm start` (или `node src/server.js`).
5. Рекомендуется добавить исключения в антивирус для папки `data\` и каталога библиотеки с книгами — иначе возможны долгие задержки и блокировки файлов.
6. За обратным прокси с HTTPS включите `TRUST_PROXY=true` и `SESSION_SECURE_COOKIE=true`, см. `.env.example`.

## Резервное копирование

- Остановите сервер (или скопируйте файл при отсутствии записи).
- Сохраните каталог `data/`, в первую очередь `data/library.db` и при необходимости `data/.session-secret`, `data/converted-books/`.
- Каталог с книгами (библиотека на диске) хранится отдельно — его бэкап по вашему сценарию.

## Обновление версии

1. Сделайте резервную копию `data/`.
2. Обновите код (git pull или распаковка релиза).
3. В корне проекта: `npm install` (или `npm ci` при фиксированном lockfile).
4. Запустите сервер; миграции SQLite выполняются при старте в `initDb()`.

Админское обновление через ZIP (`/api/operations/update`) заменяет файлы приложения и перезапускает процесс — используйте только доверенные архивы и при наличии бэкапа.

## Отдельная индексация без HTTP

Чтобы не делить event loop с веб-запросами (например, по расписанию в ночи):

```bash
npm run index:worker
# или полная пересборка:
npm run index:worker -- --full
```

Не запускайте одновременно с основным сервером, если оба пишут в одну БД (индексатор уже занят).

## Индексация при сбое БД

Если SQLite сообщает о повреждении, сначала сохраните копию `data/library.db`, затем проверьте целостность (например, `sqlite3 library.db "PRAGMA integrity_check;"`). Скрипт `npm run repair:metadata` и документация в `scripts/repair-metadata.js` — по сценарию проекта.

## Контент библиотеки

Ответственность за легальность и лицензии книг лежит на владельце развёртывания.

## OPDS и клиенты

Пути `/opds/*` и параметры поиска стараемся сохранять совместимыми между релизами; при ломающих изменениях они указываются в changelog релиза.
